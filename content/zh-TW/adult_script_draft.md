## 台詞 ID 命名規則

- 全部小寫 ASCII 加底線，穩定不隨排序改變；一個 ID 即為未來 YAML key，也是音檔檔名（例如 `s6_encourage_v03.wav`）。
- 結構：`<狀態或分類>_<用途>_<版本/序號>`
  - 狀態前綴：`s0`–`s7` 對應 FSM S0–S7；元台詞用 `meta_`；FAQ 用 `faq_`。
  - 版本後綴：`c` ＝ canonical（該狀態必講句，唯一）；`vNN` ＝ variant（語意等價變體，兩位數序號，如 `v01`、`v02`）；FAQ 答句用 `_ans`，其對應意圖（學員問法）記於備註。
  - 範例：`s2_addr_confirm_c`、`s6_encourage_v03`、`meta_clarify_01`、`faq_aed_ans`。
- 新增變體只加新序號，不重排既有 ID，確保音檔與紀錄可長期對應。

## 審定方式建議

請維護者對每一列在「狀態」欄標記處置：

- ✅ 通過（原文直接定稿）
- ✏️ 修改（請於備註或旁批附上建議改法）
- ❌ 刪除（不採用此句）

未標記者預設視為「待審」。canonical 句若需改動，請保留原 ID、只改台詞全文，以維持 ID 穩定。
