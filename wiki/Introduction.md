# 專案介紹

## 為什麼做這個工具

CPR 課程教會學員壓胸技術，但真實 OHCA（到院前心跳停止）現場，報案者一定會撥打 119——**與派遣員的對話品質直接影響 OHCA 辨識速度與旁觀者 CPR 啟動時間**（T-CPR，telecommunicator-assisted CPR，是存活鏈的關鍵一環；AHA 標準：辨識 OHCA <90 秒、電話指導下首次按壓 <150–180 秒）。傳統課堂缺乏「跟 119 對話」的體驗，本工具補上這一塊：學員面對無意識假人，實際打一通模擬的 119。

## 使用情境

- **場地**：課堂，講師的 Mac 一台（接麥克風／喇叭），學員輪流上場。
- **流程**：輸入學員代號 → 選擇情境 → AI 派遣員開場：「你好，這裡是天才消防局，請問你要消防車還是救護車？」→ 依 protocol 引導（確認地點、意識、呼吸 → 辨識 OHCA → 指導壓胸）→ 講師示意「救護人員抵達」→ 結束、顯示該學員時間指標。
- **課末**：講師按「結束課堂」→ 全班 Excel 報表 + debriefing 回饋 dashboard。

## 核心架構（一句話 + 圖）

本機網頁 app + 有限狀態機對話引擎（LLM 僅做意圖理解）+ 全預錄台詞語音 + on-device 語音辨識。

```
學員語音 → on-device STT (zh_TW) → LLM 意圖分類（回傳 JSON）
        → FSM 決定下一狀態與台詞 → 播放預錄音檔（500–800ms 內回應）
```

## 五個關鍵設計決策

1. **狀態機主導，不用純 LLM 對話**：T-CPR protocol 本身就是決策樹，真實派遣員照卡片唸。腳本化＝訓練一致性、可考核、臨床內容由醫師審定、回覆可全預錄。LLM 只負責「聽懂」學員千變萬化的說法。
2. **語音 100% 預錄**：台詞有限可窮舉，用既有 Gemini TTS pipeline（台灣腔）離線批次合成，播放零延遲。動態句已設計性繞開（地址不複誦）。
3. **語音辨識 on-device**：macOS 26 SpeechAnalyzer/SpeechTranscriber 原生支援 zh_TW，免費、零網路、內建斷句。
4. **意外輸入五層防禦**：跳步（FSM 多 slot）→ 元台詞拉回 → FAQ 庫 → 受約束即時生成（禁止新醫療指示）→ 講師安全網。生成內容課後審核轉正，系統越用越聰明。
5. **供應商抽象層**：STT／LLM／TTS 三個介面，雲端→地端切換＝改一行設定；最終目標全離線課堂。

## 教材依據

- 公開 toolkit（主 repo `docs/reference/`）：CPR LifeLinks Toolkit (2023)、Resuscitation Academy T-CPR Toolkit——皆為公開發佈之第三方教材，不在本專案 MIT 授權範圍內。
- 另有派遣 protocol 逐步腳本（FSM 與台詞的直接來源）與 T-CPR QI 表單（時間指標欄位定義來源）：屬第三方內部文件，**不隨本 repo 散布**，開發時由維護者本地提供。
