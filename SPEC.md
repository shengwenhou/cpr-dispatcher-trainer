> 定案日：2026-07-03。本文件是所有開發工作的依據；與其他文件矛盾時以本文件為準。

## 一、產品定位

- 使用者：CPR 課程學員（剛學完壓胸技術）；操作者：課堂講師（急診醫師）。
- AI 扮演 119 派遣員，依 T-CPR protocol 以繁體中文語音引導學員完成模擬報案，全程對照真實派遣流程。
- 課堂情境固定：無意識假人、無 DNR/TOR 情境、compression-only（**不做人工呼吸**）。
- 結束條件：學員說出「救護人員已抵達」類訊息（時機由現場講師示意掌控，app 不內建抵達計時）。
- 開場白固定：**「你好，這裡是天才消防局，請問你要消防車還是救護車？」**（說消防車 → 禮貌告知本版本不支援、引導回救護車）。

## 二、已定案架構決策

| 項目 | 決策 | 備註 |
|---|---|---|
| 平台 | 本機網頁 app：Python FastAPI 後端 + 瀏覽器前端，跑在講師 Mac 的 localhost | `start.command` 雙擊啟動；未來上雲/iOS 重用同一後端 |
| STT | macOS 26 SpeechAnalyzer/SpeechTranscriber（on-device, zh_TW）為首選，經由約百行 Swift helper 接入；備援 whisper.cpp large-v3-turbo | **第一個 spike：實測壓胸數數階段的辨識行為** |
| 對話引擎 | 有限狀態機（FSM）主幹 + LLM 僅做意圖分類/slot-filling（回傳 JSON, constrained decoding） | 明確不採純 LLM 驅動 |
| TTS | 100% 預錄：台詞庫以 Gemini 3.1 Flash TTS（Charon + 台灣腔 style prompt，重用既有 pipeline）離線批次生成 | 動態句已設計性繞開；即時 fallback 用 AVSpeechSynthesizer zh-TW |
| LLM | 起步：Gemini Flash-Lite API（每場 <NT$1）；v2.0 切地端 ollama + Qwen3-4B | 三個 Provider 抽象介面（STT/LLM/TTS），切換=改一行設定 |
| Metrics | FSM 狀態轉移點打 monotonic 時間戳，每筆附觸發原句 | 對齊 AHA T-CPR 標準與 QI 表單欄位 |
| 版本管理 | 本 repo + GitHub；雲端同步資料夾不放程式碼 | 金鑰一律不進 repo |

## 三、使用者介面規格（課堂多人模式）

1. **情境選單**（開始畫面）：
   - a. 成人 ／ b. 小兒 >8 歲 ／ c. 小兒 1–8 歲 ／ d. <1 歲嬰兒 ／ e. 懷孕婦女 ／ f. 其他進階情境
   - **V1.0–V2.0 僅實作「成人」**，其餘五項顯示但反灰不可選（介面預留）。
   - 架構要求：「情境」為 FSM 引擎的參數；未來開放新情境＝新增一套台詞庫＋腳本分支，不動核心。
2. **學員代號**：每位學員練習前輸入代號（同一台 Mac 輪流上場）；一場練習＝一個 session 綁定該代號。
3. **練習畫面**：即時逐字稿、目前狀態指示（供講師看）、緊急中止鈕。
4. **每場結束**：即時顯示該學員個人指標；可匯出個人報告（Word）。
5. **「結束課堂」按鈕**：
   - 產出**全班 Excel**（xlsx，可存檔）：各學員 × 各指標。
   - 顯示**debriefing 結案回饋 dashboard**：全班狀態視覺化（辨識 OHCA 時間、開始按壓時間、達標狀況、分布比較、fallback 觸發統計），供課末帶討論。
6. 資料模型：`Class（課堂）→ StudentSession（多場，各綁學員代號）→ Events（時間戳事件流）`。

## 四、對話引擎

### FSM 骨架（成人情境）

```
S0 開場（固定語音：「你好，這裡是天才消防局，請問你要消防車還是救護車？」）
S1 確認救護車需求（消防車 → 禮貌引導回，停留此狀態）
S2 確認地點 → 確認句定稿：「好的，地址我記下了，預估5分鐘之後會抵達。」（不複誦地址）
S3 確認意識（叫得醒嗎？有反應嗎？）
S4 確認呼吸（正常呼吸？聽起來怎樣？→ 瀕死喘息判斷）→【★辨識 OHCA 時間戳】
S5 指導擺位（平躺、跪旁邊、掌根置兩乳頭連線中點、雙手交疊）
S6 指導壓胸 →【★首次按壓時間戳＝首次偵測到學員數數】
   （跟著數 1-2-3、節奏 100–120/min、用力壓、不要停）
   ※ 此階段不做嚴格輪流對話：計時器每 15–20 秒插播鼓勵/糾正（預錄變體輪替），直到聽見結束訊號
S7 學員說「救護人員到了」→ 稱讚 + 宣布接手 + 結束 → 產出報告
```

- 意圖分類 JSON 一次可回傳多個 slot → FSM 支援跳步（學員一句話講完意識+呼吸+已開始壓，直接跳到第一個未填 slot）。
- LLM 信心不足 → FSM 不前進，播該狀態的預錄澄清句。

### 意外輸入五層防禦（罐頭語料不足時的處理，已與使用者確認）

| 層 | 機制 | 語音來源 |
|---|---|---|
| 1 | 跳步＝FSM 正常能力（多 slot 一次填） | 預錄 |
| 2 | 每狀態配備元台詞：釐清／承接+拉回／安撫+接管主導（連續 2 次 unknown 時） | 預錄（承接前綴+狀態問句可拼接） |
| 3 | 課堂 FAQ 意圖庫 10–20 條（AED、翻身、壓斷肋骨、嘔吐…），答完自動接回當前問題；**清單由使用者定義** | 預錄 |
| 4 | 受約束即時生成：LLM 生成短回應（禁止新醫療指示、僅安撫/承接/拉回、限字數）+ 即時 TTS；先播預錄思考語掩飾延遲 | 即時（1–2s） |
| 5 | 安全網：技術故障提示講師介入；沉默用分級 timeout reprompt | 預錄 |

- 第 4 層每次生成（原句+回應+音檔）都存檔記錄 → 課後由使用者審核 → 好的轉正為 FAQ 台詞（重跑批次合成入庫）→ 預錄覆蓋率隨課堂數收斂（目標 95%+）。
- 第 4 層不碰醫療指示內容 → 計時關鍵路徑（OHCA 辨識、按壓指導節點）永遠是固定腳本，可考核性不受影響。
- 所有 fallback 觸發記入學員紀錄與 dashboard（debriefing 素材）。

## 五、台詞規則

- 台詞由 Claude 依 `docs/reference/` protocol 起草（台灣派遣員口吻繁中）→ **使用者（急診醫師）逐句審定** → 才跑 Gemini TTS 批次合成。
- 已定稿句：
  - 開場：「你好，這裡是天才消防局，請問你要消防車還是救護車？」
  - 地址確認：「好的，地址我記下了，預估5分鐘之後會抵達。」
- 台詞庫三層：canonical（必講句）／variants（每狀態 2–4 個等價變體，鼓勵語輪替）／FAQ 答句。
- TTS style prompt 重用維護者既有的台灣腔設定與發音校正庫（「醫師」末字全聲等，存於私有區）。

## 六、Time Metrics

| 指標 | 定義 | 參考標準 |
|---|---|---|
| 辨識 OHCA 時間 | 進入 S5 − 通話開始 | AHA T-CPR <90s |
| 開始按壓時間 | S6 首次偵測數數 − 通話開始 | AHA <150–180s |
| EMS 抵達時間 | S7 觸發 − 通話開始 | 課堂由講師掌控 |
| 各階段停留時間 | 逐狀態 | 教學回饋 |
| 對話占比 | 學員/系統/沉默時間 | 診斷用 |

- monotonic clock；每筆事件附觸發原句（可回溯）。
- 「開始按壓」為語音佐證之 self-report（教學足夠）；客觀化（音訊節奏分析/假人感測）列 v1.2+。

## 七、延遲預算

預錄命中路徑（目標 ~95% 輪次）：端點偵測 300–600ms + 意圖分類 150–400ms + 取檔播放 <60ms ≈ **500–800ms**。第 4 層生成路徑 ≤2s（前置 filler 掩飾）。

## 八之一、多語言預留（i18n 設計紀律，v1.0 起生效）

v3.0 規劃支援日／泰／印尼／馬來／英語（評估見 wiki Roadmap）。為避免屆時重構，**v1.0 起即遵守**：

1. 所有使用者可見字串（派遣員台詞、UI 文字、報告文案）一律外部化為資源檔（台詞庫 YAML／UI strings 檔），程式碼**禁止 hardcode**。
2. FSM 引擎以 `locale` 為參數（與「情境」參數同層級）；語言選單於 UI 預留（V1 僅繁中可選，比照情境選單反灰模式）。
3. 語音資產 cache key 含 `locale` 欄位；`TTSProvider` 支援按語言路由不同引擎（如馬來語 fallback 傳統 Cloud TTS `ms-MY`）。
4. 各語言上線前需通過該語言的 STT spike（比照 zh-TW 流程）與當地醫療專業審定。
